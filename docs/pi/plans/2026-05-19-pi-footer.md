# pi-footer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `@fgladisch/pi-footer`, a minimal configurable Pi extension that replaces the footer with model, thinking level, project, branch, and a trailing blank line.

**Architecture:** The extension loads partial config from `~/.pi/agent/footer.json`, merges valid fields with defaults, and installs a custom `ctx.ui.setFooter()` renderer on UI session start. Runtime state stores current thinking level and the active footer render handle so branch/model/thinking changes can request re-render.

**Tech Stack:** TypeScript, Jest, Pi ExtensionAPI, `@earendil-works/pi-tui` `truncateToWidth`, Node `fs/promises`, `path`, `os`.
